Instructions:
Click the Download button to receive a CSV file that contains data on all crimes that occurred within the city of St. Louis dating back to January 2021--the point at which all US law enforcement agencies were required to switch from the SRS to the NIBRS crime reporting system. The change in reporting systems also means that comparison to crime rates prior to 2021 is not recommended without detailed knowledge of the differences between them.

Background:
I was working with a St. Louis non-profit that needed crime data within a specific geographic area in St. Louis. Unfortunately, the crime data that is made available by SLMPD is limited: there are PDFs with monthly totals by neighborhood and category, as well as CSVs with more detailed monthly "updates". But the CSVs are a mix of new crimes committed in the current month, as well as updates/changes to cases that occurred in previous months, which is only useful if you have all of the data from previous months. In order to use the data, I needed to create a comprehensive database that would automatically add new crime data and update previous crimes as more information was reported. Since I felt that this was data others might find useful, I decided to make the entire database available to the public.

Data Overview:
The downloadable CSV includes all reported crime incidents in the city of St. Louis (does not include St. Louis county). The raw data comes from the St. Louis Metropolitan Police Department, which posts monthly updates here. This database updates automatically as new data is made available, though it may take a few days for the updates to be reflected since the web scraper only checks the SLMPD site on a weekly basis. See the Data Sheet for a detailed breakdown of what the csv contains. The CSV download currently excludes all crime incidents that are determined by the SLMPD to be "unfounded." However, the database still tracks all unfounded incidents in a separate table, so if there is interest I can include a way to access that data as well.

Technical Breakdown:
This is an open-source app written in Python. For a complete walk-through of the code, check out the GitHub repo. The app exists on an AWS EC2 instance. It uses Flask to execute the backend code, an SQLite database to store the data, and Bootstrap to style the frontend. Clicking the download button above triggers a call to the database. The data is then processed and given to the user as a CSV.
